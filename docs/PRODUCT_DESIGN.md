## 1. 产品概述

### 1.1 产品定位

**合一记账** 是一款纯本地运行的个人财务管理应用，核心能力是 **导入多渠道账单、自动去重分类、生成月度账本**。用户无需手动逐条记账，只需导入各平台的账单文件，即可自动获得一份统一、清晰、分类合理的个人账本。

### 1.2 核心价值主张

| 价值 | 说明 |
|------|------|
| **一站式导入** | 支持支付宝、微信、银行、电商等主流平台账单 |
| **自动处理** | 去重 + 分类 + 汇总，无需手动操作 |
| **纯本地** | 所有数据存储在设备本地，不上传服务器 |
| **灵活查询** | 按月浏览、按时间范围搜索、按分类筛选 |
| **Windows优先** | 专为Windows优化的桌面端应用 |

### 1.3 设计原则

- **隐私优先**: 数据不离开设备，所有处理在本地完成
- **开箱即用**: 导入即出结果，减少用户配置
- **渐进披露**: 初次使用极简，高级功能按需展开

### 3.2 账单导入

#### 3.2.1 支持的数据源

| 来源 | 文件格式 | 编码 | 导出路径 |
|------|----------|------|----------|
| **支付宝** | CSV | GBK / UTF-8 | 支付宝APP → 我的 → 账单 → ··· → 开具交易流水证明 |
| **微信** | CSV | UTF-8 | 微信 → 我 → 服务 → 钱包 → 账单 → 常见问题 → 下载账单 |
| **中国银行** | XLSX | UTF-8 | 手机银行 → 账户管理 → 交易明细 → 导出 |
| **工商银行** | XLSX / CSV | GBK | 手机银行 → 明细 → 导出 |
| **建设银行** | XLSX | UTF-8 | 手机银行 → 账户 → 明细 → 导出 |
| **招商银行** | XLSX | UTF-8 | 手机银行 → 账单 → 导出发票 |
| **淘宝/天猫** | CSV / XLSX | GBK | 我的淘宝 → 我的订单 → 批量导出 |
| **京东** | XLSX | UTF-8 | 我的 → 我的订单 → 导出订单 |
| **拼多多** | CSV | UTF-8 | 拼多多 → 订单 → 导出 |
| **通用 CSV** | CSV | 自动检测 | 任意标准格式的 CSV 文件 |

#### 3.2.2 导入流程

1. 用户选择数据源类型
2. 从系统文件选择器选择文件
3. 自动解析并预览前 10 条记录
4. 用户确认映射关系（首次可手动调整列映射）
5. 执行导入 → 去重 → 分类

### 3.3 自动去重

同一笔交易可能在多个平台都有记录（如用支付宝付款的淘宝订单，在支付宝和淘宝都有记录），系统通过以下规则识别重复：

| 匹配规则 | 权重 | 说明 |
|----------|------|------|
| 时间相同 + 金额相同 + 方向相同 | 90% | 精确匹配 |
| 时间相近(±5s) + 金额相同 | 80% | 高频场景 |
| 商户名相同 + 金额相同 + 时间相近(±1min) | 70% | 模糊匹配 |
| 时间相近(±1h) + 金额相同 + 分类相同 | 50% | 低置信度 |

**处理策略**: 权重 ≥ 80% 自动合并；50%-80% 标记为疑似重复，用户确认后合并；< 50% 保留。

### 3.4 自动分类

#### 3.4.1 分类体系

```
支出
├── 餐饮         (外卖、餐厅、零食、饮品)
├── 购物         (网购、日用品、服饰、数码)
├── 交通         (打车、公交地铁、加油、停车)
├── 居住         (房租、水电煤、物业、维修)
├── 娱乐         (游戏、电影、会员订阅、旅游)
├── 医疗         (药品、门诊、体检)
├── 教育         (课程、书籍、考试)
├── 通讯         (话费、网费、软件订阅)
├── 社交         (红包、转账、聚餐)
├── 金融         (手续费、利息、保险)
└── 其他         (无法分类)

收入
├── 工资         (月薪、奖金)
├── 兼职         (副业、freelance)
├── 投资         (理财收益、股票)
├── 退款         (退货退款)
├── 转账收入     (他人转账)
└── 其他
```

#### 3.4.2 分类方式

1. **规则引擎**: 关键词匹配（如"美团"→餐饮-外卖，"滴滴"→交通-打车）
2. **商户库**: 内置常见商户分类映射表（2000+ 商户）
3. **用户学习**: 用户修改分类后记录偏好，后续相同商户自动应用
4. **置信度展示**: 高置信度自动分类，低置信度标记"待确认"

### 3.5 账本浏览

| 视图 | 说明 |
|------|------|
| **月视图** | 按月展示收支汇总 + 每日流水，顶部显示月度总收入/总支出/结余 |
| **日视图** | 按日展开，同一天内按时间排序 |
| **分类视图** | 按分类汇总，点击分类查看该分类下的所有交易 |
| **时间搜索** | 选择起止日期范围，跨月查询 |

### 3.6 统计分析

- **分类饼图**: 月度支出按分类占比
- **趋势折线图**: 近 6 个月收支趋势
- **月度对比**: 本月 vs 上月 vs 去年同期
- **消费排行**: 最高消费 Top 10 商户/商品

## 5. 用户流程

### 5.1 首次使用流程

```
启动 App
  → 欢迎页 (介绍纯本地隐私特性)
  → 创建第一个账本
  → 引导导入账单 (选择来源 → 选择文件 → 预览确认)
  → 自动去重 + 分类
  → 展示月度账本
```

### 5.2 日常使用流程

```
启动 App → 查看本月账本 → 有新账单则导入 → 查看统计
```

### 5.3 导入流程

```
点击"导入"
  → 选择数据来源 (支付宝/微信/银行卡/电商/通用CSV)
  → 系统文件选择器选择文件
  → 后台解析 (显示进度)
  → 预览解析结果 (展示前 10 条 + 统计信息)
  → 确认导入
  → 执行去重 → 执行分类
  → 显示导入报告: 新增 N 条, 跳过重复 M 条, 待确认 K 条
  → 返回账本
```

| sub_category | TEXT | INDEX | 二级分类 (如 "外卖") |

| source_transaction_id | TEXT | INDEX | 来源平台的交易单号 |
| payment_method | TEXT | - | 支付方式 |
| status | TEXT | - | 交易状态 |
| account | TEXT | INDEX | 所属账户 |
| is_duplicate | INTEGER | INDEX | 是否标记为重复 (0/1) |
| duplicate_of | INTEGER | INDEX | 指向被保留的交易 ID |

| is_confirmed | INTEGER | INDEX | 用户是否已确认分类 (0/1) |
| raw_json | TEXT | - | 原始行数据的 JSON |

| 字段 | 类型 | 说明 |
|------|------|------|

| 字段 | 类型 | 说明 |

| 字段 | 类型 | 说明 |

| total_expense | REAL | 总支出 |
| balance | REAL | 结余 |
| category_breakdown | TEXT | JSON: 各分类金额 |
| source_breakdown | TEXT | JSON: 各来源金额 |
| updated_at | TEXT | 最后更新时间 |

### 9.2 统一中间格式 (RawTransaction)

```typescript
interface RawTransaction {
  datetime: Date;           // 交易时间
  amount: number;           // 金额 (正数)
  direction: 'income' | 'expense';  // 收入/支出
  description: string;      // 商品说明
  counterparty?: string;   // 交易对方
  paymentMethod?: string;  // 支付方式
  sourceId?: string;       // 来源平台交易号
  status?: string;         // 交易状态
  source: string;          // 数据来源标识
  raw: Record<string, any>; // 原始字段 (完整保留)
```

### 9.3 各平台解析要点

#### 支付宝 CSV 解析

- 编码: GBK (需尝试 GBK → UTF-8 回退)
- 文件头: 前 N 行为元数据行 (非 CSV header)，需跳过
- 金额列: "金额(元)" 为纯数字，"收/支" 列决定方向
- 时间格式: "YYYY-MM-DD HH:MM:SS"
- 交易状态过滤: 仅导入 "交易成功" 的记录

#### 微信 CSV 解析

- 编码: UTF-8 (可能带 BOM)
- 金额列: 部分版本用 +/- 号区分收支
- 时间格式: "YYYY-MM-DD HH:MM:SS"
- 文件头: 可能包含元数据行

#### 银行 XLSX 解析

- 格式: .xlsx 使用 exceljs 库解析
- 收支列: 银行通常分 "收入金额" 和 "支出金额" 两列
- 余额列: "本次余额" (可用于数据校验)
- 日期格式: 可能为 "YYYYMMDD" 或 Excel 日期序列号

#### 电商订单解析

- 淘宝/天猫: CSV 或 XLSX，需映射 "订单金额"、"实付金额"
- 京东: XLSX，注意 "商品名称" 列用于分类
- 拼多多: CSV，需处理订单状态

## 10. 去重引擎

### 10.1 去重流程

```
新导入 RawTransaction[]
       │
       ▼
┌──────────────────┐
│  精确去重          │  sourceTransactionId 完全匹配
│  (sourceId 匹配)  │  → 直接跳过
└────────┬─────────┘
        │
        ▼
┌──────────────────┐
│  模糊去重          │  时间窗口 ±5s + 金额相等
│  (多维度匹配)     │  商户名 + 金额 + 时间窗口 ±1min
└────────┬─────────┘
        │
        ▼
┌──────────────────┐
│  跨平台去重        │  支付宝付款的淘宝订单 vs 淘宝订单
│  (支付↔订单关联)   │  时间窗口 ±1h + 金额匹配
└────────┬─────────┘
        │
        ▼
    输出去重结果
    ├── 新增 (confidence < 50%)
    ├── 跳过重复 (confidence >= 80%)
    └── 待确认 (50% <= confidence < 80%)
```

### 10.2 去重算法核心

```typescript
function calculateDuplicateScore(
  candidate: RawTransaction,
  existing: Transaction
): number {
  let score = 0;

  // 1. 精确匹配: sourceTransactionId
  if (candidate.sourceId && 
      candidate.sourceId === existing.sourceTransactionId) {
    return 1.0;  // 100% 确定重复
  }

  // 2. 时间差
  const timeDiff = Math.abs(
    candidate.datetime.getTime() - existing.datetime.getTime()
  ) / 1000;
  if (timeDiff <= 5) score += 0.4;
  else if (timeDiff <= 60) score += 0.25;
  else if (timeDiff <= 3600) score += 0.1;

  // 3. 金额匹配
  if (Math.abs(candidate.amount - Math.abs(existing.amount)) < 0.01) {
    score += 0.35;
  }

  // 4. 方向匹配
  if (candidate.direction === existing.direction) score += 0.1;

  // 5. 商户名相似度
  if (candidate.counterparty) {
    const similarity = stringSimilarity(
      candidate.counterparty,
      existing.counterparty || ''
    );
    score += similarity * 0.15;
  }

  return score;
}
```

## 12. 安全与隐私

### 12.1 数据安全措施

| 措施 | 说明 |
|------|------|
| **纯本地存储** | 所有数据存储在设备本地 SQLite 数据库，不联网、不上传 |
| **备份加密** | 导出备份文件使用 AES-256 加密，密码由用户设定 |
| **文件沙箱** | 导入的账单文件处理后立即从应用缓存清除 |
| **内存安全** | 敏感数据处理完成后立即释放，不残留在内存 |
| **无网络请求** | 应用不包含任何网络请求功能 |

### 12.2 权限声明

| 权限 | 用途 |
|------|------|
| 存储空间读取 | 读取用户选择的账单文件 |
| 存储空间写入 | 导出账本和备份文件 |

### 12.3 数据处理生命周期

```
用户选择文件 → 读取到内存 → 解析为 RawTransaction → 写入数据库
                                              → 清除内存中的原始数据
                                              → 清除临时文件缓存
```
