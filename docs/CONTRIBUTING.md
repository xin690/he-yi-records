# 贡献指南

欢迎为合一记账贡献力量！

## 如何贡献

### 报告 Bug

1. 前往 [Issues](https://github.com/xin690/he-yi-records/issues)
2. 创建新 Issue，选择 Bug 模板
3. 描述问题并提供复现步骤

### 提出新功能

1. 前往 [Issues](https://github.com/xin690/he-yi-records/issues)
2. 创建新 Issue，选择 Feature Request 模板
3. 描述功能需求和使用场景

### 提交代码

1. Fork 本仓库
2. 创建功能分支 (`git checkout -b feature/your-feature`)
3. 进行开发并提交
4. 推送分支 (`git push origin feature/your-feature`)
5. 创建 Pull Request

## 开发环境

### 前置要求

- Node.js >= 18
- npm >= 9

### 本地开发

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

### 代码规范

- 使用 ESLint 检查代码
- 使用 Prettier 格式化代码
- 提交前运行 `npm run lint`

## 行为准则

- 请保持友好和尊重
- 欢迎各种形式的贡献，无论大小
- 在提交 Issue 或 PR 前，请先搜索是否已有类似内容

---

感谢您的贡献！
