## 2. 构建APK/AAB

### 2.1 构建命令

```bash
# 步骤1: 构建Web资源
npm run build

# 步骤2: 同步到Android平台
npx capacitor sync android

# 步骤3: 进入Android目录
cd android

# 步骤4: 构建Debug APK (测试用)
./gradlew assembleDebug

# 步骤5: 构建Release AAB (上架用)
./gradlew assembleRelease
```

### 2.2 输出产物

| 类型        | 路径                                                       | 说明   |
| ----------- | ---------------------------------------------------------- | ------ |
| Debug APK   | `android/app/build/outputs/apk/debug/app-debug.apk`        | 测试用 |
| Release AAB | `android/app/build/outputs/bundle/release/app-release.aab` | 上架用 |

### 2.3 签名配置

- 当前使用: 已有签名密钥
- 密钥位置: `android/app/key.jks` (或配置的密钥)
- 如需创建新密钥:
  ```bash
  keytool -genkey -v -keystore key.jks -keyalg RSA -keysize 2048 -validity 10000 -alias my-key
  ```

## 3. Play Console 配置

### 3.1 创建应用

1. 登录 [Play Console](https://play.google.com/apps/publish)
2. 点击「创建应用」
3. 填写信息:
   - 应用名称: 合一记账
   - 默认语言: 简体中文
   - 应用类型: 应用
4. 点击「创建」

### 3.2 上传APK/AAB

1. 进入「发布」→ 「正式版」
2. 点击「创建新的发布」
3. 上传 AAB 文件
4. 填写版本信息

## 4. Data Safety 填写

### 4.1 访问路径

Play Console → 应用 → 隐私权保护 → Data Safety

### 4.2 填写内容

| 问题              | 答案                       |
| ----------------- | -------------------------- |
| 是否收集用户数据? | 是，处理用户导入的账单文件 |
| 数据收集方式?     | 用户主动导入               |
| 是否分享给第三方? | 否                         |
| 数据存储位置?     | 本地设备                   |
| 数据加密?         | SQLite本地存储             |
| 用户能否删除数据? | 是，卸载应用即可删除       |

### 4.3 权限声明

| 权限     | 用途说明                    |
| -------- | --------------------------- |
| 存储权限 | 从设备导入CSV/Excel账单文件 |

## 6. 提交审核

### 6.1 审核前检查

- [ ] APK/AAB 已上传
- [ ] Data Safety 已填写
- [ ] 应用信息已完善
- [ ] 图标和截图已上传
- [ ] 隐私政策链接已填写

### 6.2 提交审核

1. 进入「发布」→ 「正式版」
2. 点击「查看发布前报告」(可选)
3. 点击「开始发布」
4. 确认发布

### 6.3 审核时间

- 通常: 24-48小时
- 复杂情况: 7天

## 7. 常见问题

### Q1: APK vs AAB 选择哪个?

> AAB (Android App Bundle) 是Google推荐格式，体积更小，Play自动生成优化后的APK。

### Q2: 审核被拒绝常见原因?

> - 权限说明不清晰
> - 隐私政策链接无效
> - Data Safety信息不完整
> - 应用功能与描述不符

### Q3: 如何更新版本?

> 1. 修改 `android/app/build.gradle` 中的 `versionCode` 和 `versionName`
> 2. 重新构建 Release AAB
> 3. 在 Play Console 上传新版本

### Q4: 应用需要哪些权限?

> 当前仅需存储权限(读取导入账单)，无需网络权限。
